# Neutral Yb Atlas 设计规格

## 1. 产品定位

Neutral Yb Atlas 是面向中性镱原子量子计算研究者的双语科研知识系统。它同时服务两类用户：

- 新进入该领域的研究者，需要沿严格依赖关系理解量子计算、中性原子平台和 Yb 实现；
- 已从事理论或实验工作的研究者，需要快速检索模型、参数、实验工具、标定方法、证据等级和开放问题。

产品形态采用“科研图谱 + 实验工作台”，并保留连续阅读模式。它不是营销网站、论文列表或通用 CMS。

## 2. 成功标准

1. 首次进入者可以沿七段主线从量子信息基础走到 Yb 容错架构。
2. 专业用户可以在三次交互内定位一个公式、实验参数、仪器类别或证据来源。
3. 关键参数与结论具有证据状态，不把教学近似、候选数据和论文确认值混为一谈。
4. 中英文切换不改变信息架构，术语、公式和变量保持一致。
5. 物理参数、图表、笔记和正文均可编辑；修改保存在本地，并可导入导出。
6. 站点可作为纯静态项目部署到 GitHub Pages，不依赖私有后端或密钥。
7. 桌面端呈现专业工作台密度，移动端保留核心阅读、参数调节和笔记功能。

## 3. 信息架构

### 3.1 七段物理主线

1. 量子计算底层：态、测量、幺正演化、纠缠、门、噪声和纠错。
2. 中性原子底层：激光冷却、偶极阱、光镊阵列、重排与 Rydberg 相互作用。
3. 为什么选择 Yb：同位素、核自旋、钟态、窄线跃迁、擦除偏置及跨平台比较。
4. 从原子到量子门：选择定则、Rabi 动力学、blockade、条件相位、CZ 与单比特门。
5. 实验系统：真空、原子源、激光、光学、成像、控制、同步、标定与验收。
6. 理论服务实验：哈密顿量、主方程、参数反演、误差预算、最优控制和模型验证。
7. 走向容错：擦除转换、中电路测量、逻辑通道、QEC 接口、扩展挑战与研究路线。

### 3.2 横向研究入口

- 参数与常数索引；
- 仪器与工具索引；
- 公式和模型索引；
- 论文与证据索引；
- 开放问题与挑战索引；
- 实验故障诊断索引。

### 3.3 阅读模式

- `Guided Path`：显示依赖关系、先修知识和阶段进度；
- `Research Reference`：弱化教学提示，强化目录、筛选、证据状态和交叉引用；
- `Workbench`：集中呈现模型参数、图表、实验流程、笔记和导出操作。

## 4. 首页与章节结构

### 4.1 首屏

- 安静的顶部导航：品牌、四个核心入口、中英文切换、编辑按钮和研究索引；
- 左侧为标题、简短定义和两个明确入口；
- 右侧为“公理层 → 物理层 → 器件层 → 系统层”的知识依赖主轴，不在首屏放置完整能级参考图或参数探针；
- 首屏底部露出下一段物理主线，避免封闭式全屏英雄区；
- 首屏不使用眉题、徽章、虚假指标或营销式卖点。

完整 171Yb 能级参考图位于 Yb 原子结构章节，在交互能级教学图之后呈现。第一遍学习使用分层、可选择的教学图；高密度参考图只承担总览、交叉核对和原始分辨率查看。

允许的首屏文字：

- `Neutral Yb Atlas`；
- `中性镱原子计算的理论与实验基础`；
- 对应英文标题；
- 一段不超过 70 个中文字的定义；
- `进入物理主线`、`打开参数工作台`；
- 导航名称和模式控制。

### 4.2 章节页面

每章保持一致的研究结构，但视觉节奏可变化：

1. 本章物理问题；
2. 最小模型与第一性原理推导；
3. 关键实验可观测量；
4. 模型边界和常见误解；
5. 可交互图或参数实验；
6. 实验实现和标定；
7. 证据、引用和延伸问题。

章节不使用重复卡片网格。主要容器是开放版面、横向带、参数表、时间线、画布和侧栏。

## 5. 核心内容范围

### 5.1 基础层

- Hilbert 空间、复振幅、Born 规则、密度矩阵；
- 单比特、Bloch 球、复合系统、纠缠；
- 门模型、模拟量子计算、测量、噪声和量子纠错；
- 为什么量子优势不是“并行尝试所有答案”。

### 5.2 中性原子层

- 原子能级、光与物质相互作用、Rabi 振荡和旋转波近似；
- Doppler/亚 Doppler 冷却、MOT、偶极势和光镊；
- 随机装载、成像、重排、运动和阵列几何；
- Rydberg 态、van der Waals/偶极相互作用和 blockade 半径。

### 5.3 Yb 平台层

- 171Yb、174Yb 等同位素角色；
- 1S0、3P0、3P1、1P1 和目标 Rydberg 态；
- 399 nm、556 nm、578 nm、302 nm 及相关探测/擦除通道；
- 地态核自旋与亚稳态量子比特方案；
- Yb 与 Rb、Cs、Sr 在理论、实验和工程上的归一化比较；
- 擦除转换、中电路检测和读出隔离的价值与成本。

### 5.4 门与误差层

- 单原子和两原子哈密顿量；
- blockade CZ、连续相位/振幅/失谐调制；
- 平均门保真度、条件相位、泄漏、损失和 SPAM；
- 自发辐射、Doppler、激光相位/强度噪声、指向误差、串扰和有限 blockade；
- Lindblad、Monte Carlo、滤波函数和响应感知脉冲优化；
- 从物理门通道到擦除/Pauli/泄漏逻辑通道的映射。

### 5.5 实验层

- 建设阶段、前置依赖、周期范围和并行任务；
- 每阶段的工具、关键规格、输入输出接口、标定方法和验收条件；
- 真空、原子源、磁场、激光、倍频、锁频、AOM/EOM、AOD/SLM、显微物镜、相机、FPGA/AWG 和时钟同步；
- 日常运行 SOP、漂移监测、数据记录、风险和故障树；
- 采购信息仅提供规格维度和选择原则，不把易变化的具体价格写死。

## 6. 理论工作台

### 6.1 交互模型

- Rabi 振荡；
- 两原子 blockade 动力学；
- CZ 相位与泄漏；
- Doppler 与温度影响；
- 误差预算；
- 激光噪声谱与滤波函数重叠；
- 擦除/Pauli 通道映射。

所有模型提供默认参数、单位、取值范围、证据状态和“恢复默认值”。参数变化即时更新图表，但不把教学模型显示为实验预测。

### 6.2 参数快照

用户可保存命名快照，内容包括：

- 模型 ID 和版本；
- 参数值和单位；
- 证据状态；
- 用户笔记；
- 创建时间；
- 内容语言不影响参数键名。

## 7. 实验工作台

实验流程采用依赖时间线，而非静态器件列表：

1. 真空与原子源；
2. 冷却/成像激光；
3. MOT 与光镊装载；
4. 阵列成像和重排；
5. 量子比特制备与相干标定；
6. Rydberg 光谱与 blockade；
7. 门、读出、擦除检测和基准测试。

每个阶段包含：目标、依赖、工具、关键规格、校准、验收指标、常见失败、输出记录和预计周期。周期为范围值，并明确受采购、加工和实验室条件影响。

## 8. 编辑与本地数据

### 8.1 编辑范围

- 物理参数和图表输入；
- 个人笔记和高亮；
- 章节正文和图注；
- 参数快照与实验清单状态。

### 8.2 持久化

- 使用版本化 `localStorage` 保存轻量数据；
- 正文修改按内容块 ID 保存，不直接改写源文件；
- 导出统一 JSON；
- 笔记可额外导出 Markdown；
- 导入前执行 schema 校验并显示差异摘要；
- 数据迁移失败时保留原始备份，不静默丢弃。

### 8.3 开源边界

浏览器本地修改不会自动上传 GitHub。公开贡献通过正常的仓库修改和 Pull Request 完成，避免在纯静态站点中伪造协作后端。

## 9. 双语策略

- 内容数据使用稳定键，中文和英文为并列字段；
- 公式、符号、参考文献和参数键共享；
- 语言切换保持滚动位置、展开状态、输入值和笔记；
- 首次访问根据浏览器语言选择，之后遵循用户设置；
- 不采用逐字直译，英文使用该领域标准术语，中文首次出现时给出英文术语。

## 10. 证据体系

参数状态：

- `confirmed`；
- `confirmed_from_official_source_data`；
- `derived_confirmed`；
- `candidate`；
- `candidate_digitized`；
- `missing`；
- `unavailable`。

结论边界：

- 教学模型；
- 预测性基准；
- 参数锁定基准；
- Methods 级复现；
- 研究级鲁棒重拟合。

每个关键参数显示来源、单位、适用模型、证据状态和最后核验日期。候选值不得以相同视觉权重冒充确认值。

## 11. 视觉系统

### 11.1 设计方向

现代科学出版物与实验仪器界面的结合。页面的动态感来自物理状态变化、轨迹、时间线、滚动关系和工作台状态，而不是装饰动画。

### 11.2 色彩

- 背景：冷白 `#f8f9f6` 和真白 `#ffffff`；
- 正文：石墨黑 `#17201f`；
- 仪器工作台：深绿 `#123f39`；
- 关键实验节点：朱红 `#b34d34`；
- 302 nm/Rydberg：紫色 `#7047a8`；
- 399 nm：蓝色 `#2f6ca3`；
- 实时控制与选中状态：受控使用黄色 `#e7b948`。

颜色必须承担语义，不能形成单一色相主题。

### 11.3 排版

- 标题使用科研出版物风格衬线字体；
- 正文与控制使用高可读无衬线字体；
- 公式使用 KaTeX/兼容数学字体；
- 字号不随视口宽度连续缩放；
- 所有 letter-spacing 为 0；
- 控件、侧栏、表格和图例具有独立明确的字号层级。

### 11.4 容器与图形

- 圆角不超过 4 px；
- 不使用卡片套卡片；
- 不使用 bento 网格、渐变球、光晕、霓虹网格或装饰性大圆；
- 原子、能级、光束、脉冲、阵列和数据图是主要视觉资产；
- 教学图必须同时包含对象、关系、关键变量、实验映射、图例或证据边界；装饰图和实验室照片不能替代教学结构；
- 图标使用 Lucide 线性图标，并统一尺寸和笔画；
- 关键科研图尽量采用可交互 SVG/Canvas，复杂实验场景可使用授权图片或原创位图。

### 11.5 动效

- 首屏能级和光束对参数变化作轻微响应；
- 章节进入时只做小幅位移和透明度变化；
- 图表更新使用 150–250 ms 过渡；
- 实验时间线强调当前阶段及依赖；
- 尊重 `prefers-reduced-motion`。

## 12. 响应式设计

### 桌面端

- 三段首屏：章节轨、正文、原子图；
- 理论工作台使用侧栏 + 图表 + 参数面板；
- 实验时间线横向展开。

### 平板端

- 首屏正文与原子图上下排列；
- 工作台侧栏折叠为顶部标签；
- 时间线两列排列。

### 移动端

- 固定底部章节导航；
- 原子图保留关键能级，详细标注进入全屏图谱；
- 参数控件单列且具有稳定高度；
- 表格切换为字段行，不横向压缩正文；
- 正文编辑采用全屏模式；
- 不允许横向溢出或控制遮挡内容。

## 13. 技术架构

- React + TypeScript + Vite；
- 纯静态构建，兼容 GitHub Pages 子路径；
- 内容与组件分离，双语内容采用类型化数据模块；
- KaTeX 渲染公式；
- Recharts 或轻量 SVG 实现数据图；
- Lucide React 提供通用图标；
- 本地状态使用 React 状态与版本化存储适配器；
- 不引入后端、数据库、账户系统或分析 SDK。

主要模块：

- `AppShell`：导航、语言、模式、编辑入口；
- `KnowledgeMap`：七段主线和章节依赖；
- `ArticleRenderer`：双语内容、公式、引用和可编辑块；
- `AtomicMap`：Yb 能级和光参数；
- `TheoryWorkbench`：模型、参数和图表；
- `ExperimentWorkbench`：流程、工具、验收和诊断；
- `EvidenceRegistry`：参数与声明边界；
- `LocalWorkspace`：笔记、正文覆盖、快照和导入导出。

## 14. 错误处理

- 内容数据加载失败：显示可定位的模块错误，不让整站空白；
- 公式解析失败：保留源公式并标注渲染错误；
- 导入数据无效：拒绝写入并显示字段级问题；
- 本地存储不可用：退化为内存会话并提示导出；
- 图表参数超界：钳制到物理范围并显示原因；
- 外部引用不可访问：保留书目信息，不隐藏论据边界。

## 15. 验证策略

- 单元测试：单位转换、blockade 比值、模型输入、存储迁移和双语键完整性；
- 组件测试：语言切换、编辑、导入导出、参数联动和证据筛选；
- 构建检查：TypeScript、ESLint、生产构建和 GitHub Pages 基路径；
- 浏览器检查：桌面、平板、移动端首屏与核心流程；
- 视觉检查：概念稿与实现截图逐段比较；
- 科学检查：公式、单位、参数状态和引用的抽样交叉核验；
- 无障碍：键盘访问、焦点、对比度、标签和减少动态效果。

## 16. 非目标

- 不实现在线协同编辑；
- 不实现用户账户或云同步；
- 不提供生产实验设备的自动控制；
- 不把候选模型包装成实验结论；
- 不复制受版权限制的论文全文；
- 不以大量动画、3D 场景或营销式视觉牺牲科研阅读效率。

## 17. 已知约束

- 当前会话中的内置图像生成功能不可用；视觉概念使用交互 HTML 完成。
- 浏览器策略当前阻止自动访问视觉伴侣本地地址，因此概念稿尚未完成自动截图验收。
- 正式实现需要在浏览器访问条件允许后补齐桌面/移动端截图比较与交互测试。
